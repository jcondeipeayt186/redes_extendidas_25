Capa 6 - Presentación

Formato y Encriptación de Datos

La capa de presentación se encarga de la representación de datos, formateo y encriptación para asegurar la compatibilidad entre sistemas.

  2. Protocolos

¿Qué es la Capa de Presentación?

La Capa 6 del modelo OSI se encarga de la representación de datos, formateo y encriptación para asegurar la compatibilidad entre sistemas.

Función Principal: Asegurar que los datos se presenten en un formato que la capa de aplicación pueda entender.

Responsabilidades de la Capa de Presentación

Formateo de datos
Convertir datos a formatos estándar
Encriptación
Proteger datos sensibles
Compresión
Reducir el tamaño de los datos
Traducción
Convertir entre diferentes codificaciones
Verificación
Validar integridad de datos
Conversión
Adaptar formatos entre sistemas

Protocolos de la Capa 6

SSL/TLS

Secure Sockets Layer / Transport Layer Security:

  • Encriptación de datos
  • Autenticación de servidores
  • Integridad de datos
  • Certificados digitales
Compresión

Algoritmos de compresión:

  • GZIP
  • JPEG, PNG, GIF
  • MPEG, AVI
  • ZIP, RAR
Codificación

Estándares de codificación:

  • ASCII
  • Unicode (UTF-8, UTF-16)
  • Base64
  • MIME
Conversión

Servicios de conversión:

  • EBCDIC a ASCII
  • Big-endian a Little-endian
  • Formatos de fecha
  • Unidades de medida
