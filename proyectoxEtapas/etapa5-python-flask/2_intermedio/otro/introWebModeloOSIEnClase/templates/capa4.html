Capa 4 - Transporte

Control de Flujo y Confiabilidad

La capa de transporte se encarga de la entrega confiable de datos entre aplicaciones y proporciona control de flujo y corrección de errores.

¿Qué es la Capa de Transporte?

La Capa 4 del modelo OSI se encarga de la entrega confiable de datos entre aplicaciones y proporciona control de flujo y corrección de errores.

Función Principal: Garantizar la entrega confiable y ordenada de datos entre aplicaciones de origen y destino.

Responsabilidades de la Capa de Transporte

Entrega confiable
Garantizar la entrega sin errores
Control de flujo
Regular la velocidad de transmisión
Segmentación
Dividir datos en segmentos manejables
Corrección de errores
Detectar y corregir errores de transmisión
Direccionamiento de puertos
Identificar aplicaciones específicas
Reensamblaje
Reconstruir datos en el orden correcto

Protocolos de la Capa 4

TCP (Transmission Control Protocol)

Protocolo orientado a conexión:

  • Entrega confiable y ordenada
  • Control de flujo y congestión
  • Corrección de errores
  • Puertos de origen y destino
UDP (User Datagram Protocol)

Protocolo no orientado a conexión:

  • Transmisión rápida sin garantías
  • Sin control de flujo
  • Sin corrección de errores
  • Ideal para aplicaciones en tiempo real
ARP

Address Resolution Protocol:

  • Mapeo IP a MAC
  • Tabla ARP
  • Broadcast para resolución
  • Cache local
Protocolos de Enrutamiento

Protocolos para intercambiar información de rutas:

  • RIP (Routing Information Protocol)
  • OSPF (Open Shortest Path First)
  • BGP (Border Gateway Protocol)
  • EIGRP (Enhanced Interior Gateway Routing)
