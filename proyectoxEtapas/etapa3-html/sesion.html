¿Qué hace la Capa de Sesión?

La Capa de Sesión es la quinta capa del modelo OSI y se encarga de establecer, gestionar y terminar sesiones de comunicación entre aplicaciones. Esta capa permite que dos aplicaciones establezcan una conexión lógica y mantengan el estado de la comunicación durante el tiempo que sea necesario.

Sus funciones principales incluyen el establecimiento de sesiones, la sincronización de datos, el control de diálogo, la recuperación de sesiones interrumpidas, y la gestión de tokens para controlar el acceso a recursos compartidos.

Protocolos de la Capa de Sesión

NetBIOS (Network Basic Input/Output System)

Descripción: Protocolo desarrollado por IBM que permite a las aplicaciones comunicarse a través de una red local.

Función: Proporciona servicios de sesión, transporte y nombres para aplicaciones de red en sistemas Windows.

RPC (Remote Procedure Call)

Descripción: Protocolo que permite a un programa ejecutar código en otra computadora a través de la red.

Función: Establece sesiones para llamadas a procedimientos remotos, gestiona la comunicación cliente-servidor y mantiene el estado de las operaciones.

SQL*Net

Descripción: Protocolo de Oracle para establecer sesiones de base de datos a través de la red.

Función: Gestiona conexiones de base de datos, autenticación de usuarios y mantenimiento de sesiones activas.

X.225 (ISO 8327)

Descripción: Estándar ISO para protocolos de sesión que define cómo establecer y gestionar sesiones de comunicación.

Función: Proporciona servicios de sesión estandarizados, control de diálogo y sincronización de datos.

SSH (Secure Shell)

Descripción: Protocolo que proporciona acceso seguro a sistemas remotos y gestión de sesiones cifradas.

Función: Establece sesiones seguras, autentica usuarios y mantiene conexiones cifradas para operaciones remotas.

Funcionalidades de Prueba

Como usuario registrado, puedes realizar las siguientes pruebas relacionadas con la Capa de Sesión:

1. Gestión de Sesiones de Usuario

Esta prueba te permite experimentar con la gestión de sesiones:

  • Simula el inicio de sesión de un usuario
  • Observa el establecimiento de la sesión
  • Gestiona múltiples sesiones simultáneas
  • Simula el cierre y la expiración de sesiones

Protocolos involucrados: NetBIOS, SSH, RPC

Conceptos de red: Establecimiento de sesiones, autenticación, gestión de estado

2. Sincronización de Datos

Experimenta con la sincronización entre aplicaciones:

  • Simula la sincronización de archivos entre dispositivos
  • Observa el control de diálogo en comunicaciones bidireccionales
  • Analiza puntos de sincronización y recuperación
  • Experimenta con la resincronización después de interrupciones

Protocolos involucrados: RPC, X.225

Conceptos de red: Sincronización, control de diálogo, puntos de control

3. Recuperación de Sesiones Interrumpidas

Aprende sobre la robustez de las sesiones:

  • Simula interrupciones de conexión
  • Observa el proceso de recuperación automática
  • Analiza diferentes estrategias de recuperación
  • Experimenta con la reanudación de sesiones

Protocolos involucrados: SSH, RPC, X.225

Conceptos de red: Recuperación de sesiones, robustez, continuidad de servicio

Ejemplo Práctico: Sesión de Chat

Cuando mantienes una conversación por chat desde Higueras con tu amigo en Zarautz, la Capa de Sesión:

  1. Establecimiento: Se establece una sesión de chat entre tu aplicación y la de tu amigo
  2. Autenticación: Se verifica tu identidad y se crea un token de sesión
  3. Control de diálogo: Se alterna el turno para enviar mensajes (evita colisiones)
  4. Sincronización: Se mantiene sincronizado el estado de la conversación
  5. Persistencia: La sesión se mantiene activa aunque haya pausas en la conversación
  6. Recuperación: Si se pierde la conexión, se puede reanudar la sesión
  7. Terminación: Al cerrar el chat, se termina la sesión de manera ordenada

Estados de una Sesión

Una sesión típica pasa por varios estados durante su ciclo de vida:

  1. INACTIVA: No hay sesión establecida
  2. ESTABLECIENDO: Se está negociando el establecimiento de la sesión
  3. ACTIVA: La sesión está funcionando normalmente
  4. SUSPENDIDA: La sesión está pausada temporalmente
  5. RECUPERANDO: Se está recuperando de una interrupción
  6. TERMINANDO: Se está cerrando la sesión de manera ordenada
  7. CERRADA: La sesión ha terminado completamente

Servicios de la Capa de Sesión

Servicio Descripción Ejemplo de Uso
Establecimiento de Sesión Crear una conexión lógica entre aplicaciones Iniciar sesión en una aplicación web
Control de Diálogo Gestionar quién puede enviar datos y cuándo Chat en tiempo real con turnos
Sincronización Mantener datos sincronizados entre aplicaciones Sincronización de archivos en la nube
Recuperación Restaurar el estado de la sesión después de fallos Reanudar descarga interrumpida
Terminación Cerrar la sesión de manera ordenada Cerrar sesión en una aplicación
