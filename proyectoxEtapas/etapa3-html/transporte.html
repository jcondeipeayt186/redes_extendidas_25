¿Qué hace la Capa de Transporte?

La Capa de Transporte es la cuarta capa del modelo OSI y se encarga de proporcionar comunicación confiable y eficiente entre aplicaciones que se ejecutan en diferentes dispositivos. Esta capa garantiza que los datos lleguen completos, en el orden correcto y sin errores.

Sus funciones principales incluyen el control de flujo, la detección y corrección de errores, el control de congestión, la segmentación y reensamblaje de datos, y la multiplexación de múltiples aplicaciones en una sola conexión de red.

Protocolos de la Capa de Transporte

TCP (Transmission Control Protocol)

Descripción: Es un protocolo orientado a conexión que garantiza la entrega confiable de datos en el orden correcto.

Función: Establece conexiones virtuales, controla el flujo de datos, detecta y corrige errores, y reensambla los segmentos en el orden correcto.

UDP (User Datagram Protocol)

Descripción: Es un protocolo no orientado a conexión que proporciona transmisión rápida pero no garantiza la entrega ni el orden.

Función: Envía datagramas independientes sin establecer conexión previa, ideal para aplicaciones que requieren velocidad sobre confiabilidad.

SCTP (Stream Control Transmission Protocol)

Descripción: Protocolo moderno que combina las mejores características de TCP y UDP, diseñado para aplicaciones de telecomunicaciones.

Función: Proporciona transmisión confiable con múltiples streams, detección de errores y control de congestión.

DCCP (Datagram Congestion Control Protocol)

Descripción: Protocolo que proporciona control de congestión para aplicaciones que no requieren entrega confiable.

Función: Combina la simplicidad de UDP con el control de congestión de TCP, ideal para aplicaciones multimedia en tiempo real.

Funcionalidades de Prueba

Como usuario registrado, puedes realizar las siguientes pruebas relacionadas con la Capa de Transporte:

1. Comparación TCP vs UDP

Esta prueba te permite experimentar con las diferencias entre TCP y UDP:

  • Simula transmisión de datos con TCP (orientado a conexión)
  • Simula transmisión de datos con UDP (no orientado a conexión)
  • Observa el establecimiento de conexión TCP (3-way handshake)
  • Compara velocidades y confiabilidad de ambos protocolos

Protocolos involucrados: TCP, UDP

Conceptos de red: Orientación a conexión, control de flujo, confiabilidad

2. Análisis de Puertos y Conexiones

Explora el sistema de puertos y conexiones:

  • Visualiza puertos bien conocidos (0-1023)
  • Simula el establecimiento de conexiones TCP
  • Analiza el estado de conexiones (ESTABLISHED, LISTENING, etc.)
  • Experimenta con multiplexación de aplicaciones

Protocolos involucrados: TCP, UDP

Conceptos de red: Puertos, conexiones, multiplexación, estados de conexión

3. Control de Flujo y Congestión

Aprende sobre mecanismos de control:

  • Simula control de flujo con ventana deslizante
  • Observa algoritmos de control de congestión
  • Analiza el comportamiento de TCP en redes congestionadas
  • Experimenta con diferentes tamaños de ventana

Protocolos involucrados: TCP

Conceptos de red: Control de flujo, control de congestión, ventana deslizante

Ejemplo Práctico: Transmisión de un Mensaje

Cuando envías un mensaje desde Higueras a Zarautz, la Capa de Transporte:

  1. Segmentación: Tu mensaje se divide en segmentos de tamaño manejable
  2. Establecimiento de conexión: TCP establece una conexión virtual con tu amigo en España
  3. Transmisión: Los segmentos se envían uno por uno con números de secuencia
  4. Control de flujo: Se ajusta la velocidad de transmisión según la capacidad del receptor
  5. Detección de errores: Se verifica la integridad de cada segmento recibido
  6. Reensamblaje: Los segmentos se reensamblan en el orden correcto
  7. Confirmación: Se confirma la recepción exitosa del mensaje completo

Los puertos del 0 al 1023 están reservados para servicios estándar:

Estados de Conexión TCP

Una conexión TCP pasa por varios estados durante su ciclo de vida:

  1. CLOSED: No hay conexión activa
  2. LISTEN: El servidor espera conexiones entrantes
  3. SYN_SENT: El cliente ha enviado SYN, espera respuesta
  4. SYN_RECEIVED: El servidor ha recibido SYN, enviado SYN+ACK
  5. ESTABLISHED: Conexión establecida, intercambio de datos
  6. FIN_WAIT_1: Aplicación ha cerrado, enviado FIN
  7. FIN_WAIT_2: FIN enviado, esperando FIN del otro lado
  8. CLOSE_WAIT: FIN recibido, esperando que la aplicación cierre
  9. LAST_ACK: Enviado FIN, esperando ACK
  10. TIME_WAIT: Esperando tiempo para asegurar que ACK llegue
