¿Qué hace la Capa de Enlace de Datos?

La Capa de Enlace de Datos es la segunda capa del modelo OSI y se encarga de la transmisión confiable de datos entre nodos adyacentes en una red. Esta capa toma los bits de la capa física y los organiza en tramas (frames) que contienen información de control para la detección y corrección de errores.

Sus funciones principales incluyen el control de acceso al medio físico, la detección de errores, el control de flujo, y la identificación de dispositivos mediante direcciones MAC (Media Access Control).

Protocolos de la Capa de Enlace

Ethernet (IEEE 802.3)

Descripción: Es el estándar más utilizado para redes locales. Define cómo se organizan los datos en tramas y cómo se accede al medio compartido.

Función: Controla el acceso múltiple al medio, detecta colisiones y organiza los datos en tramas con direcciones MAC.

WiFi (IEEE 802.11)

Descripción: Estándar para redes inalámbricas que incluye protocolos de enlace específicos para el medio inalámbrico.

Función: Gestiona el acceso al medio inalámbrico, controla la calidad de la señal y maneja la autenticación.

PPP (Point-to-Point Protocol)

Descripción: Protocolo de enlace punto a punto utilizado principalmente para conexiones de acceso a internet.

Función: Establece conexiones directas entre dos nodos, autentica usuarios y negocia parámetros de conexión.

HDLC (High-Level Data Link Control)

Descripción: Protocolo de enlace de alto nivel utilizado en redes WAN y conexiones seriales.

Función: Proporciona control de flujo, detección de errores y control de secuencia para transmisiones confiables.

ATM (Asynchronous Transfer Mode)

Descripción: Protocolo de enlace que transmite datos en celdas de tamaño fijo, utilizado en redes de alta velocidad.

Función: Organiza los datos en celdas de 53 bytes, garantiza calidad de servicio y maneja diferentes tipos de tráfico.

Funcionalidades de Prueba

Como usuario registrado, puedes realizar las siguientes pruebas relacionadas con la Capa de Enlace:

1. Análisis de Tramas Ethernet

Esta prueba te permite examinar la estructura de las tramas de red:

  • Visualiza la estructura de una trama Ethernet
  • Identifica campos como dirección MAC origen y destino
  • Analiza el campo de datos y CRC
  • Simula la creación de tramas personalizadas

Protocolos involucrados: Ethernet (IEEE 802.3)

Conceptos de red: Estructura de tramas, direcciones MAC, detección de errores

2. Simulación de Control de Acceso al Medio

Experimenta con diferentes métodos de acceso al medio:

  • CSMA/CD (Ethernet con cable)
  • CSMA/CA (WiFi inalámbrico)
  • Token Ring (paso de testigo)
  • Análisis de colisiones y resolución

Protocolos involucrados: Ethernet, WiFi, Token Ring

Conceptos de red: Acceso múltiple, colisiones, backoff exponencial

3. Detección y Corrección de Errores

Aprende sobre mecanismos de detección de errores:

  • Cálculo de CRC (Cyclic Redundancy Check)
  • Verificación de paridad
  • Detección de errores de transmisión
  • Simulación de errores y su detección

Protocolos involucrados: Ethernet, HDLC, PPP

Conceptos de red: Códigos de detección de errores, integridad de datos

Ejemplo Práctico: Transmisión de una Trama

Cuando envías un mensaje desde Higueras a Zarautz, la Capa de Enlace:

  1. Recibe datos: Los bits de la capa física se organizan en tramas
  2. Agrega encabezado: Se incluye dirección MAC origen (tu computadora) y destino (próximo router)
  3. Control de errores: Se calcula el CRC para detectar errores de transmisión
  4. Control de flujo: Se verifica que el receptor pueda procesar la trama
  5. Transmisión: La trama se envía al siguiente nodo de la red
  6. Verificación: El receptor verifica el CRC y confirma la recepción

Direcciones MAC

Las direcciones MAC son identificadores únicos de 48 bits (6 bytes) asignados a cada interfaz de red:

Formato Ejemplo Descripción
XX:XX:XX:XX:XX:XX 00:1B:44:11:3A:B7 Formato hexadecimal con dos puntos
XX-XX-XX-XX-XX-XX 00-1B-44-11-3A-B7 Formato hexadecimal con guiones
XXXXXXXXXXXX 001B44113AB7 Formato hexadecimal continuo
